## 提示词盒子 v1.0.5b 重大 bug 修复 (09:18-09:27)

### 根因发现：esc() 函数从未定义！
- 全代码 20+ 处调用 `esc(s)` 来转义 HTML（防 XSS），但函数体从未被定义
- 可能是某次重构时被误删
- 导致所有涉及 `esc()` 的函数静默失败：
  - `openM()` → `esc(f.name)` 构建文件夹下拉 → 报错，modal 无法打开
  - `render()` → `esc(p.title)` 渲染卡片 → 部分报错
  - 任何涉及用户内容的 HTML 渲染都可能炸

### 修复方法
- 添加 `esc()` 函数：`s.replace(/&/g,'&amp;').replace(/</g,'&lt;').replace(/>/g,'&gt;').replace(/"/g,'&quot;').replace(/'/g,'&#39;')`
- 用浏览器 + Puppeteer 实测 `openM()` 验证修复有效

### 经验教训
- **未定义函数在 inline onclick 中不会报错**，因为错误被静默吞掉了
- 必须用 `try-catch` 包裹或在控制台 `evaluate()` 调用才能发现
- 构建/发布流程需要加入 lint 检查，防止类似遗漏

### timeoutSeconds
- 再次调整：1200s → 1800s (30分钟)

## 提示词盒子项目总结 — 经验教训

### CSS/HTML 踩坑
- `env(safe-area-inset-*)` Android WebView 不支持，用 JS 注入状态栏高度补偿
- CSS 变量未定义时引用不会报错，静默失效，极难排查（如 --bg2-rgb）
- `\u00D7` 是 JS Unicode 转义，HTML 正文里必须用 `&#215;` 实体
- `rgba(color, opacity)` 叠在背景上的混合色 ≠ 原色，需计算
- `theme-color` meta 标签颜色需和实际渲染色一致，不能直接用 CSS 变量色

### Android WebView
- 键盘覆盖模式：`position: fixed` 不会抬升，需监听 visualViewport.resize + translateY
- 小米 HyperOS 深色模式：标准 API 不生效，需反射调用 Window.setStatusBarDarkMode()
- `fitsSystemWindows(true)` 是正确姿势，false 会导致布局错位
- GitHub API 对 `.github/workflows/` 文件 PUT 返回 400/404，需手动网页操作

### 开发流程
- 写含代码的大文件不要用 bash heredoc，特殊字符会截断，用 python3 分段写入
- 子代理处理复杂任务可能不完整，需主 agent 兜底
- ⚠️ **应该加入自动化 UI 测试**，不能让用户帮忙测 bug
- GitHub API 推大 base64 要 120s 超时
- 状态栏适配经验：混合色计算 + meta theme-color 同步
